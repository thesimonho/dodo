# References

Default directory: `docs/`

These are random notes and md files of information that might be useful to other contributors on the project. Could be information about different parts of the project, or just a list of links to other resources.

Could be TODOs, could be plans. Varies massively by project and personal preference.

## Create

This is not generally something you would create by yourself, and the user is the one that should determine what is/isnt a useful reference

## Update

This is one of the MOST important things to keep up to date for the user.

Do not let references go stale. Project requirements change, new features are added, etc.
